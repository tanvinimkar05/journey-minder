Web Sandbox Constraints & Alternatives

Web browsers operate inside strict sandboxes. Building a background stop-alert application directly in a mobile web browser presents distinct challenges. Below are the restrictions and how native alternatives solve them:

Autoplay & Mute Control Sideloading

Web Restriction: Browsers restrict audio playback unless triggered by direct user interaction first. Furthermore, browsers cannot bypass physical hardware mute toggles (silent mode) or sleep-mode audio restrictions.

Native Solution:

Native apps (Java/Kotlin, Swift) access custom audio channels. They utilize the Alarm Audio Stream (which bypasses silent mode) and request Critical Alarm Permissions to ring at maximum volume even on Do Not Disturb (DND).

Background GPS Suspensions

Web Restriction: If a mobile user locks their screen or switches tabs, the browser immediately throttles Javascript execution. Geolocation tracking halts, and timers suspend, making the app fail to alert you.

Native Solution:

Native platforms support Foreground Services (Android) and Background Location Updates (iOS) with geofencing. The OS actively wakes up the app when the user enters the predefined GPS radius, consuming minimal battery.

Reliable Haptic & Vibration Loops

Web Restriction: While `navigator.vibrate` exists in HTML5, it is heavily restricted or entirely unsupported (e.g., iOS Safari completely blocks vibration). It also cannot trigger in the background.

Native Solution:

Native APIs tap directly into the phone's tactile vibration engine (`VibratorManager` on Android and `UIFeedbackGenerator` on iOS), enabling custom pulsing rhythms that penetrate deep sleep.

Native Wrapper Alternative (Capacitor/Cordova)

To deploy this app onto app stores, one should wrap it in a native shell. Below is a code architecture showing how the app triggers a System Alarm Notification bypassing Web limitations:

// Native Capacitor Bridge for Background Alarm Alert
import { LocalNotifications } from '@capacitor/local-notifications';

async function triggerBypassingAlarm(stopName) {
  await LocalNotifications.schedule({
    notifications: [{
      title: "TransitWake ALERT!",
      body: `Wake Up! ${stopName} is approaching!`,
      id: 101,
      sound: "critical_siren.wav", // Placed in native assets
      channelId: "critical-alerts", // Android critical audio channel
      ios: {
        critical: true, // Bypass iOS ring/silent switch
        volume: 1.0     // Max volume
      },
      vibration: [0, 500, 200, 500, 200, 1000] // Powerful pulse
    }]
  });
}
